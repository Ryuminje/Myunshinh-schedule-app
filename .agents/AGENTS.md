# 스마트 수업교체 도우미 (Schedule App) 개발 및 유지보수 가이드

이 문서는 '스마트 수업교체 도우미' 프로젝트의 개발 원칙, 아키텍처 및 주요 비즈니스 로직을 기록한 프로젝트 수준의 규칙(AGENTS.md)입니다. 에이전트가 이 프로젝트를 수정하거나 확장할 때 반드시 참고해야 합니다.

## 1. 기술 스택 (Tech Stack)
- **프레임워크**: Next.js (App Router), React
- **언어**: TypeScript
- **스타일링**: Tailwind CSS (전역 색상: 그린 파스텔 톤, 둥근 모서리 디자인 `rounded` 우선 적용)
- **아이콘**: lucide-react
- **데이터 파싱**: xlsx (SheetJS) - 서버 사이드에서 구글 시트 파싱

## 2. 데이터 흐름 (Data Flow)
- **데이터 소스**: 구글 스프레드시트 (URL 쿼리를 통해 전체 문서를 xlsx 형태로 다운로드)
- **API 라우트 (`src/app/api/schedule/route.ts`)**: CORS 문제를 방지하기 위해 Next.js 서버 사이드에서 구글 시트(`xlsx`)를 fetch 및 파싱하여 프론트엔드로 전달합니다. 
- **전역 상태 관리 (`src/lib/ScheduleContext.tsx`)**: 클라이언트는 API를 호출하여 시간표(`ScheduleData`)를 가져오고, 앱 전체에 Context API로 상태를 공급합니다.
- **캐싱 정책**: 구글 시트의 최신 데이터를 즉각 반영하기 위해 fetch 시 `{ cache: 'no-store' }` 옵션을 사용합니다.

## 3. 핵심 비즈니스 로직

### A. 교체 시간표 찾기 (`SwapTab.tsx`)
- 특정 선생님의 수업(셀)을 클릭하면, **교체 가능한 다른 선생님**을 찾아줍니다.
- **1순위 (동과 대강)**: 선택한 수업과 학년/반이 같은 수업을 가진 다른 교사 (맞교환).
- **2순위 (일반 교체)**: 선택한 시간에 수업이 없고(공강), 내가 교체해 줄 시간에 공강인 교사 (대강).
- 선택 시 전체 시간표 중 대상이 되는 교사의 행(Row)만 필터링되어 화면에 남으며, 대상 셀에 하이라이트가 표시됩니다.

### B. 협의회 시간 찾기 (`MeetingTab.tsx`)
- 다수의 교사를 선택하고 요일/교시 필터를 적용하여 **모두가 공강인 시간**을 계산합니다.
- '설정' 시트의 E~F열(협의회 불가 요일/교시)은 글로벌 블록(Global Meeting Block)으로 취급되어 해당 시간은 공통 공강 계산에서 제외됩니다.

### C. 교체 불가 설정 (`BlockTab.tsx`)
- **구글 시트 고정 설정**: 구글 시트의 '설정' 탭(A~C열)에서 읽어온 데이터입니다. 요일/교시가 비어있는 경우 무시(No Block)하며, 특정 요일/교시가 지정되면 교체 대상에서 원천적으로 제외됩니다. 앱 내에서는 삭제가 불가합니다.
- **브라우저 임시 설정**: 당일 병가, 출장 등 갑작스러운 상황 발생 시 브라우저 내(`localStorage`)에만 임시로 저장하여 적용하는 교체 불가 목록입니다.

## 4. UI/UX 디자인 원칙
- 모든 표(Table)는 가로 스크롤을 지양하며, 화면 폭에 맞춰 꽉 차게 렌더링되도록 `table-fixed` 및 `max-w-[1920px]` 등의 속성을 활용합니다.
- 모달(Modal) 창은 화면 전체를 가리거나 클릭/스크롤을 방해하지 않도록, 우측 상단의 플로팅 패널(Floating Panel) 형태로 띄워 뒤쪽의 시간표를 직접 확인할 수 있도록 합니다.
